Office Use — Page & Data

ℹ
Who can access: SuperAdmin, Admin, Officer. Viewer is blocked. Served via ?page=office&ref=REF-YYYY-NNNN. All write ops require CAN_EDIT_ROLES on the backend.

The Office Use page is where staff process incoming applications — filling in membership details, recording fees, and moving applications through the status lifecycle.

Status lifecycle
Pending → Under Review → Approved or Rejected

Rejection requires a mandatory reason string. Approval auto-generates a MEM-YYYY-NNNN number and sends an email to the applicant.

Session & Auth Flow

STEP 01
Credentials
username + password
STEP 02
Sheet lookup
AdminUsers, Active=true
STEP 03
UUID token
sess_{uuid} stored
STEP 04
Browser store
localStorage client
STEP 05
Per-request check
validateToken() called
Session storage engine
  • Engine: ScriptProperties (Apps Script)
  • Key: sess_{uuid}
  • Value: JSON — username, role, expiry ms
  • Max concurrent sessions: 50, auto-cleaned
Token lifecycle
  • Duration: 30 min (SESSION_DURATION)
  • Sliding window — resets on each valid call
  • Logout: logoutUser(token) deletes key
  • Cleanup: cleanExpiredSessions_() on login
⚠
No top-level PropertiesService calls. Calling PropertiesService at top level crashes before OAuth completes — validateToken() always fails and pages hang on "Verifying…". All config loads lazily via getConfig_() inside each function.

Technical Reference

// Standard guard pattern — every protected backend function function someProtectedFunction(token, data) { getConfig_(); const session = checkToken_(token); if (!session.valid) return { success: false, message: 'Unauthorized' }; // Tier 1 — edit ops (SuperAdmin, Admin, Officer) if (CAN_EDIT_ROLES.indexOf(session.role) === -1) return { success: false, message: 'Permission denied.' }; // Tier 2 — manage ops (SuperAdmin, Admin) if (CAN_MANAGE_ROLES.indexOf(session.role) === -1) return { success: false, message: 'Permission denied.' }; // Tier 3 — delete only (SuperAdmin) if (CAN_DELETE_ROLES.indexOf(session.role) === -1) return { success: false, message: 'Only SuperAdmin can delete.' }; }
// ROLE_PAGE_ACCESS + permission tier arrays const ROLE_PAGE_ACCESS = { SuperAdmin: ['dashboard', 'office', 'payments'], Admin: ['dashboard', 'office', 'payments'], Officer: ['dashboard', 'office'], Viewer: ['dashboard'], }; const CAN_EDIT_ROLES = ['SuperAdmin', 'Admin', 'Officer']; const CAN_MANAGE_ROLES = ['SuperAdmin', 'Admin']; const CAN_DELETE_ROLES = ['SuperAdmin'];
// validateToken() — sliding-window session check function validateToken(token) { getConfig_(); if (!token || token.trim() === '') return { valid: false }; const props = PropertiesService.getScriptProperties(); const sessKey = SESS_PREFIX + token.trim(); const raw = props.getProperty(sessKey); if (!raw) return { valid: false }; const sess = JSON.parse(raw); if (Date.now() > sess.expiry) { props.deleteProperty(sessKey); return { valid: false }; } sess.expiry = Date.now() + CONFIG.SESSION_DURATION; // reset window props.setProperty(sessKey, JSON.stringify(sess)); return { valid: true, username: sess.username, role: sess.role }; }

Configuration & Setup

Script Properties required
  • SPREADSHEET_ID — Google Sheets ID
  • ADMIN_EMAIL — notification recipient
  • DRIVE_FOLDER_ID — signature file storage
  • PAYMENT_SLIP_FOLDER_ID — payment slips
  • LOGO_FILE_ID — org logo in Drive
Sheets required
  • Applications — 37 columns, main data
  • AdminUsers — username, password, role, active
  • AnnualPayments — 12 columns, dues tracking
  • Auto-created by setupSystem() if missing
Setup sequence
  • Run setupSystem() — creates all sheets
  • Run addFirstAdmin() — seeds default account
  • Set script properties via the editor UI
  • Run checkEmailConfig() to verify email
  • Run sendTestEmail() to confirm delivery
Default seed accounts
  • admin / Admin@1234 → SuperAdmin
  • officer / Officer@1234 → Officer
  • viewer / Viewer@1234 → Viewer
  • Change all passwords immediately after setup
⚠
Security note: Default passwords are hardcoded in addRoleAccounts(). Change immediately. AdminUsers sheet stores passwords in plain text — restrict spreadsheet sharing strictly in Google Drive.
✓
v4.9 fix: saveApplication() corrected from saveApplication(token, formData, sig) to saveApplication(formData, sig). Old signature caused formData to land in the token slot, making nameEnglish validation always fail with "nameEnglish අවශ්‍යයි".
